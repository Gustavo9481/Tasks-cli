    Pruebas: Modelo Task

    tests.models_tests.test_model_task

    Este módulo contiene las pruebas unitarias para el modelo Task.

    Tests unitarios para la clase Task. total de pruebas: 10.

    test_task_creation_with_and_without_details()

    Comprueba la creación de una instancia Tarea con y sin detalles.

    Verifica que el campo 'details' se asigna correctamente cuando se proporciona y que su valor por defecto es None cuando no se especifica.

    test_task_creation_with_defaults()

    Comprueba la creación de una instancia Tarea con valores defaults.

    El test verifica que la creación de una instancia de Task especificando sólo el valor de content(contenido de la tarea o descripción), la tarea se crea correctamente con los valores por defecto en sus otras propiedades.

    test_task_creation_with_invalid_content_type_boolean()

    Comprueba error generado con content tipo booleano.

    El test verifica que si una tarea es generada con un valor de tipo boleano para su propiedad content, no será generada.

    Raises:

    Type Description
    ValidationError

    Cuando content es de tipo booleano.

    test_task_creation_with_invalid_content_type_float()

    Comprueba error generado con content tipo float.

    El presente test verifica que si una tarea se crea usando un valor de tipo float para la propiedad content, no será creada.

    Raises:

    Type Description
    ValidationError

    Cuando content es de tipo float.

    test_task_creation_with_invalid_content_type_integer()

    Comprueba error generado con content tipo entero.

    Este test verifica que si el tipo de valor para la propiedad content resulta ser un entero, la tarea no será generada.

    Raises:

    Type Description
    ValidationError

    Cuando content es de tipo int.

    test_task_creation_with_invalid_priority()

    Comprueba error generado con valor de prioridad inválida.

    El test verifica que si la tarea contiene un valor no permitido en la propiedad priority, no será generada. Los valores permitidos están definidos en el módulo models/model_task.py en el objeto Literal Priority.

    Raises:

    Type Description
    ValidationError

    Cuando el valor de prioridad no es uno de los permitidos.

    test_task_creation_with_invalid_status()

    Comprueba error con un status inválido.

    Este test verifica que si la tarea es creada con un valor de status no válido, la tarea no se crea. Los valores válidos se definen en el módulo models/model_task.py en la lista Literal Status.

    Raises:

    Type Description
    ValidationError

    Cuando el status tiene un valor no permitido.

    test_task_creation_with_invalid_tag()

    Comprueba error con un tag inválido.

    El test verifica que si el valor de tag (etiqueta) es un valor distinto a los valores permitidos, la tarea no será generada. Los valores admitidos se definen en el módulo models/model_task.py en el Literal Tag.

    Raises:

    Type Description
    ValidationError

    Cuando tag tiene un valor no permitido.

    test_task_creation_with_missing_content()

    Comprueba error generado al no definir content.

    El test verifica que si una tarea es generada sin hacer la definición correspondiente de la propiedad content, no será generada.

    Raises:

    Type Description
    ValidationError

    Cuando no se define la propiedad content.

    test_task_creation_with_no_defaults()

    Comprueba la creación de una instancia de Tarea con valores distintos a los defaults.

    El test verifica que si la tarea se crea con valores distintos a los defaults pero aceptados en sus propiedades status, tag y priority, la tarea se crea correctamente. Los valores aceptados se especifican en models/model_task.py en las listas Literal [Status, Tag, Priority].
