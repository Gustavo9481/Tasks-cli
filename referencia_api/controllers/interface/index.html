    Controlador: Interface

    controllers.interface

    Este módulo define la clase principal de la aplicación Textual, que orquesta la UI y los eventos.

    Clase Interface

    Bases: App

    Clase principal de la interfaz Textual para la app 'Tasks-cli' de lista de tareas.

    Como clase principal, se encarga de: - Componer la layout inicial con widgets (Header, DataTable, Footer). - Definir los atajos de teclado globales (BINDINGS). - Lanzar acciones (action_*) que muestran pantallas modales para la interacción con el usuario. - Recibir los resultados de las pantallas modales y ejecutar la lógica de negocio correspondiente a través de TaskService.

    action_add_task()

    Maneja el atajo de teclado 'n' para añadir una nueva tarea.

    Esta acción abre la pantalla modal AddTaskScreen y le asigna el método notification_add_task como callback para procesar el resultado.

    action_check_or_uncheck_task()

    Maneja el atajo 'm' para marcar/desmarcar una tarea (cambiar status)

    Abre la pantalla modal AskIdScreen para solicitar el ID de la tarea y asigna notification_check_or_uncheck_task como callback.

    action_delete_task()

    Maneja el atajo 'd' para eliminar una tarea.

    Abre la pantalla modal AskIdScreen para solicitar el ID y asigna notification_delete_task como callback.

    action_edit_task()

    Maneja el atajo 'e' para iniciar la edición de una tarea.

    Abre AskIdScreen para obtener el ID de la tarea a editar y asigna _start_edit_process como callback.

    action_filter_tasks()

    Maneja el atajo 'f' para filtrar tareas.

    Abre la pantalla modal FilterTasksScreen y asigna notification_filter_tasks como callback.

    action_reset_filters()

    Maneja el atajo 'r' para limpiar filtros y refrescar la tabla.

    Llama directamente a _update_table() para recargar la lista completa de tareas.

    action_view_details()

    Maneja el atajo 'v' para ver los detalles de una tarea.

    Abre AskIdScreen para obtener el ID y asigna _show_details_screen como callback.

    compose()

    Compone el layout inicial de la aplicación.

    Este método de Textual se llama una vez al iniciar la app para renderizar los widgets estáticos como el Header, Footer y DataTable.

    notification_add_task(new_task_data)

    Callback que procesa los datos recibidos de AddTaskScreen.

    Este método es llamado por Textual cuando la pantalla AddTaskScreen se cierra. Valida los datos, utiliza TaskService para crear la tarea y actualiza la tabla visual.

    Parameters:

    Name Type Description Default
    - new_task_data dict | None

    Diccionario con los datos de la tarea enviado desde el modal. Es None si el usuario canceló la operación.

    		 required

    notification_check_or_uncheck_task(task_id)

    Callback que cambia el estado de la tarea.

    Es llamado por Textual al cerrar AskIdScreen. Utiliza TaskService para cambiar el estado de la tarea y refresca la tabla.

    Parameters:

    Name Type Description Default
    - task_id int

    ID de la tarea a modificar, validado por el decorador @require_valid_id.

    		 required

    notification_delete_task(task_id)

    Callback que elimina la tarea especificada.

    Es llamado por Textual al cerrar AskIdScreen. Emplea TaskService para eliminar la tarea y actualiza la tabla.

    Parameters:

    Name Type Description Default
    - task_id int

    ID de la tarea a eliminar, validado por el decorador @require_valid_id.

    		 required

    notification_filter_tasks(filter_data)

    Callback que recibe los criterios de filtro y actualiza la tabla.

    Limpia y estandariza los datos del filtro, llama al servicio para obtener las tareas filtradas y actualiza el DataTable con los resultados.

    Parameters:

    Name Type Description Default
    - filter_data dict | None

    Diccionario con los filtros a aplicar. Ej: {'status': 'pending', 'priority': 'alta'}. Es None si el usuario canceló la pantalla de filtros.

    		 required

    on_mount()

    Se ejecuta cuando la app se monta en el DOM.

    Este método de Textual se usa para realizar configuraciones iniciales, como definir las cabeceras de la tabla y cargar los datos por primera vez.
