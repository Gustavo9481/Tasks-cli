    Repositorio: RepositoryDB

    repositories.repository_db

    Este módulo define la capa de acceso a datos para la base de datos de tareas.

    Clase RepositoryDB

    Gestiona todas las operaciones de la base de datos para las tareas.

    Esta clase abstrae las consultas SQL y proporciona una interfaz clara para que la capa de servicio interactúe con la base de datos.

    __init__(db_name)

    Inicializa el repositorio y establece la ruta a la base de datos.

    Parameters:

    Name Type Description Default
    - db_name str

    El nombre del archivo de la base de datos (ej. "tasks-cli.db").

    		 required

    task_format_list(rows_list)

    Convierte una lista de filas de la BD en una lista de objetos Task.

    Este método auxiliar actúa como una capa de mapeo, transformando los datos crudos de la base de datos (lista de tuplas) en una lista de objetos Task validados por Pydantic.

    Parameters:

    Name Type Description Default
    - rows_list list

    Lista de filas (tuplas) obtenida de una consulta a la base de datos.

    		 required

    Returns:

    Type Description
    list[Task]
    • list[Task]: Lista de objetos Task completamente formados.

    create_table(cursor=sqlite3.Cursor)

    Asegura que la tabla 'tasks_table' exista en la base de datos.

    Ejecuta la sentencia SQL para crear la tabla si esta no existe. La gestión de la conexión y el commit es manejada por el decorador.

    Parameters:

    Name Type Description Default
    - cursor Cursor

    Cursor de la base de datos, inyectado por el decorador connection_manager.

    		 required

    get_all_tasks(cursor)

    Recupera todas las tareas de la base de datos.

    Si la tabla no existe o hay un error, devuelve una lista vacía.

    Parameters:

    Name Type Description Default
    - cursor Cursor

    Cursor de la base de datos, inyectado por el decorador.

    		 required

    Returns:

    Type Description
    list[Task]
    • list[Task]: Una lista de objetos Task que representan todas las tareas en la base de datos. Estará vacía si sqlite3 devuelve error.

    new_task(task_instance, cursor)

    Inserta una nueva tarea en la base de datos.

    Parameters:

    Name Type Description Default
    - task_instance Task

    Objeto Task con los datos a insertar.

    		 required
    - cursor Cursor

    Cursor de la base de datos, inyectado por el decorador.

    		 required

    Returns:

    Type Description
    int
    • int: ID de la fila de la tarea recién creada.

    filter_tasks(cursor, status=None, tag=None, priority=None)

    Filtra tareas dinámicamente por status, tag y/o prioridad.

    Utiliza un patrón Strategy para seleccionar la consulta SQL adecuada basándose en los filtros proporcionados. Si no se proporciona ningún filtro, devuelve todas las tareas.

    Parameters:

    Name Type Description Default
    - cursor Cursor

    Cursor de la base de datos, inyectado por el decorador.

    		 required
    - status str | None

    Estado por el cual filtrar.

    		 required
    - tag str | None

    Etiqueta por la cual filtrar.

    		 required
    - priority str | None

    Prioridad por la cual filtrar.

    		 required

    Returns:

    Type Description
    list[Task]
    • list[Task]: Lista de objetos Task que coinciden con los criterios de filtrado.

    update_task(task_id, new_data, cursor)

    Actualiza uno o más campos de una tarea existente de forma dinámica.

    Construye la sentencia SQL dinámicamente a partir de los datos proporcionados en el diccionario new_data.

    Parameters:

    Name Type Description Default
    - task_id int

    ID de la tarea a actualizar.

    		 required
    - new_data dict[str, str]

    Diccionario con los campos a actualizar y sus nuevos valores. Ejemplo: {"content": "Nuevo contenido", "priority": "alta"}

    		 required
    - cursor Cursor

    Cursor de la base de datos, inyectado por el decorador.

    		 required

    check_or_uncheck_task(id_task, cursor)

    Cambia el estado de una tarea de forma cíclica.

    Verifica primero si la tarea existe. Si existe, utiliza la consulta UPDATE_STATUS_TOGGLE para rotar el estado.

    Parameters:

    Name Type Description Default
    - id_task int

    ID de la tarea cuyo estado se va a cambiar.

    		 required
    - cursor Cursor

    Cursor de la base de datos, inyectado por el decorador.

    		 required

    get_task_by_id(id_task, cursor)

    Recupera una única tarea por su ID.

    Parameters:

    Name Type Description Default
    - id_task int

    ID de la tarea a recuperar.

    		 required
    - cursor Cursor

    Cursor de la base de datos, inyectado por el decorador.

    		 required

    Returns:

    Type Description
    Task | None
    • Task | None: Objeto Task si se encuentra la tarea, o None si no existe ninguna tarea con ese ID.
